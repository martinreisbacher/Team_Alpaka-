## HERMES-Phasen
Das Projekt folgt dem klassischen HERMES-Phasenmodell:

| Phase | Beschreibung | Wichtige Dokumente | Abgabetermin |
|-------|--------------|--------------------|------------- |
| Initialisierung | Klärung von Zielen, Stakeholdern, Risiken und Rahmenbedingungen. | Projektinitialisierungsauftrag, Stakeholderliste, PMP. Dokumente 1-10 | 08.03.2026 |
| Konzept | Ausarbeitung der Lösung, Anforderungen und Machbarkeitsprüfung. | Situationsanalyse, Testkonzept, Integrationskonzept. | 14.03.2026 |
| Realisierung | Entwicklung, Tests und Integration des Produkts. | Code-Docs, Testprotokolle. |
| Einführung | Rollout in den Betrieb, Schulung und Übergabe. | Einführungsplan, Schulungsmaterial. |
| Abschluss | Finale Abnahme, Lessons Learned und Projektende. | Abschluss-Projektauftrag. |
| | | |
|  | Phasenübergreifende Dokumente | Protokolle,Arbeitsplan,Projektentscheidungen Dokumente ab 31 |
